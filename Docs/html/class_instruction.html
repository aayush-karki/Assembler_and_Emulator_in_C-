◆ OpCode

enum class Instruction::OpCode
strong

Op code are the intrusction that are to be performed.

There are two different type of Op codes. First type of Op codes that are not supported in the machine language, but are used by the assembler. (ORG, END, DC, DS) And Second type of Op codes are supported in both machine language and assembler language. These Op codes have a corresponding number that are used by the machine language.

When using Op codes, they can be all upper cased, or lowered cased, or a combination of both

Note
Addr is a label in assembley language and a location in machine language
when used in machine languge numbers are used to represent both the op codes and label/location
c(Addr) is the content that label represents in assembley language or stored in that location in memory
read AddrA <– c(AddrB) + c(AddrC) as storing the final result of operation in right of '<–' (here, c(AddrB) + c(AddrC) ) in to the location or label in the left
number of label/location right of <– determines the number of label/location need for the particular Op code

HALT 

usage: HALT
Terminate execution. The Addr1 and Addr2 are ignored

ORG 

usage: ORG 100
define origin.
The operand specifies the address at which the translation of the next instruction will be generated
Here, next intruction starts at location 100

DC 

usage: Addr1 DC 5
Defines constant.
The constant is a decimal integer placed in the operand field
Here, c(Addr1) = 5

DS 

usage: Addr1 DS 3
Defines storage.
The operand specifies the number of words of storage to be set aside
Here, Addr1 points to the start of a array of length 3

END 

usage: END
Indicates that there are no additional statements to translate.
